Git使用教程

Git使用教程

Git是什么

Git是目前世界上最先进的分布式版本控制系统。
工作原理 / 流程：

Workspace：工作区
Index / Stage：暂存区
Repository：仓库区（或本地仓库）
Remote：远程仓库

安装Git

首先在官网下载Git，默认安装完成后，在开始菜单找到“Git –> Git Bash”,打开后输入

因为Git是分布式版本控制系统，所以需要填写用户名和邮箱作为一个标识。

注意：git config –global 参数，有了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置，当然你也可以对某个仓库指定的不同的用户名和邮箱。

一、创建版本库

什么是版本库？版本库又名仓库，英文名repository,你可以简单的理解一个目录，这个目录里面的所有文件都可以被Git管理起来 。

在任何一个文件夹打开Git bash, 创建一个版本库

创建完后可以在 testgit 目录下新建一个文件 test.txt, 然后可以使用一下命令提交文件到仓库

1.初始化仓库

git init

2.将文件添加到暂存区

git add test.txt

3.把文件提交到仓库

git commit -m "first commit"

提交后可以查看是否还有文件未提交：git status

也可以修改test文件后查看修改内容：git diff test.txt

二、版本回退

查看文件修改历史记录：git log

如果嫌显示信息太多，也可以使用命令 git log -pretty=online

版本回退命令：

1.回退到上一个版本

git reset --hard HEAD^

2.回退到上上一个版本

git reset --hard HEAD^^

3.回退到前100个版本

git reset --hard HEAD~100

4.回退到特定版本

git reset --hard 版本号

ps: 可以使用 git reflog 查询版本号

三、工作区与暂存区区别

工作区：就是你在电脑上看到的目录，比如目录下testgit里的文件(.git隐藏目录版本库除外)。或者以后需要再新建的目录文件等等都属于工作区范畴。
版本库(Repository)：工作区有一个隐藏目录.git,这个不属于工作区，这是版本库。其中版本库里面存了很多东西，其中最重要的就是stage(暂存区)，还有Git为我们自动创建了第一个分支master,以及指向master的一个指针HEAD。

四、撤销修改和删除文件

一、撤销修改

当某个文件修改后，但是还没有提交，想要撤销修改时，可以采用一下方法：

1.如果我知道要删掉那些内容的话，直接手动更改去掉那些需要的文件，然后add添加到暂存区，最后commit掉。

2.直接恢复到上一个版本。使用 git reset --hard HEAD^

3.使用命令：git checkout -- test.txt

二、删除文件

1.直接手动删除

2.rm filename

如果向彻底从文件库删除，可以执行 commit 命令提交掉

在没有提交前，可以使用git checkout -- filename 从版本库中恢复此文件

五、远程仓库

1.创建ssh key

打开命令行，输入：

ssh-keygen -t rsa –C “youremail@example.com"

然后就会在用户主目录生成.ssh文件夹，包含公钥 idrsa.pub 和私钥 idrsa

2.登录 github, 打开” settings”中的SSH Keys页面，然后点击“Add SSH Key”,填上任意title，在Key文本框里黏贴id_rsa.pub 文件的内容, 点击 Add Key。

3.添加远程库

git remote add origin repositoryname

4.把本地仓库分支master内容推送到原仓库

git push -u origin master

由于远程库是空的，我们第一次推送master分支时，加上了 –u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

5.从远程克隆仓库

git clone repostoryname

六、创建与合并分支

1.基本命令

查看分支：git branch

创建分支：it branch name

切换分支：git checkout name

创建+切换分支：git checkout –b name

合并某分支到当前分支：git merge name

删除分支：git branch –d name

2.如何解决冲突？

在冲突时查看内容，修改成和主干代码一样的

3.分支管理策略

通常合并分支时，git一般使用”Fast forward”模式，在这种模式下，删除分支后，会丢掉分支信息，可以使用带参数 –no-ff来禁用”Fast forward”模式，即git merge --no-ff dev

分支策略：首先master主分支应该是非常稳定的，也就是用来发布新版本，一般情况下不允许在上面干活，干活一般情况下在新建的 dev 分支上干活，干完后，比如上要发布，或者说 dev 分支代码稳定后可以合并到主分支master上来。

七、bug分支

1.将当前工作现场隐藏

git stash

2.在master分支上创建临时分支

git checkout -b issue-404

3.修改文件并提交

4.切换到master分支，完成合并，并删除issue-404分支

5.回到dev分支，搜索工作现场

6.恢复工作现场

首先恢复：git stash apply，然后删除stash内容： git stash drop

或者直接恢复并删除：git stash pop

八、多人协作

当你从远程库克隆时候，实际上Git自动把本地的master分支和远程的master分支对应起来了，并且远程库的默认名称是origin。

查看原仓库信息：git remote

查看远程库的详细信息：git remote -v

一、推送分支

主分支：git push origin master

其它分支：git push origin dev

二、抓取分支

当其它小伙伴对某个分支 推送了提交 ，而在我的 而我在我的目录文件下也对同样的文件同个地方作了修改，也试图推送到远程库时，就会推送失败。

因为我的小伙伴最新提交的和我试图推送的有冲突，解决的办法也很简单，先用git pull把最新的提交从origin/dev抓下来，然后在本地合并，解决冲突，再推送。

1.指定本地dev分支与远程origin/dev分支的链接

git branch --set-upstream dev origin/dev

2.抓取

git pull

3.解决冲突

4.提交再push

因此：多人协作工作模式一般是这样的：

首先，可以试图用git push origin branch-name推送自己的修改.
如果推送失败，则因为远程分支比你的本地更新早，需要先用git pull试图合并。
如果合并有冲突，则需要解决冲突，并在本地提交。再用git push origin branch-name推送。
